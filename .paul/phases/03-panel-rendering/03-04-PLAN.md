<objective>
## Goal
Implementasi auto-timeout idle: setelah `gDisplayTimeout` detik dari event terakhir, display otomatis kembali ke idle state (P1 hitam, P3 placeholder, P2/P4 normal).

## Purpose
`gDisplayTimeout` sudah disimpan di NVS dan dapat diupdate via MQTT config topic, tapi belum digunakan. Plan ini mengaktifkan timer sehingga display parking gate tidak terus-terusan menampilkan status lama setelah event selesai.

## Output
- `gLastEventMs` global — timestamp event non-idle terakhir
- `handleEventTopic()` diupdate: set `gLastEventMs` saat event non-idle diterima
- loop(): timeout check — reset ke idle setelah `gDisplayTimeout` detik
- `gDisplayTimeout = 0` menonaktifkan timeout (display tetap sampai event berikutnya)
</objective>


    After the status validation block in `handleEventTopic()` (the block that sets `gGate.status`), add:
    ```cpp
    // Track timestamp for display timeout (only non-idle events reset the timer)
    if (gGate.status != "idle") {
      gLastEventMs = millis();
    }
    ```

    Place this AFTER the `gGate.status = s` assignment and BEFORE the plate/anprStatus assignments.

    **Step 3: Add timeout check to loop()**

    In `loop()`, after the scroll tick block and BEFORE `if (redrawNeeded)`, add:
    ```cpp
    // Auto-timeout: reset to idle after gDisplayTimeout seconds from last event
    if (gGate.status != "idle" && gDisplayTimeout > 0) {
      const unsigned long nowMs = millis();
      if (nowMs - gLastEventMs >= static_cast<unsigned long>(gDisplayTimeout) * 1000UL) {
        gGate.status     = "idle";
        gGate.plate      = "";
        gGate.anprStatus = "";
        redrawNeeded     = true;
        Serial.printf("[timeout] display reset to idle after %us\n", gDisplayTimeout);
      }
    }
    ```

    Notes:
    - `gDisplayTimeout = 0` → condition `gDisplayTimeout > 0` is false → no timeout (AC-3)
    - Idle status check `gGate.status != "idle"` prevents redundant resets (AC-5)
    - `unsigned long` cast prevents overflow: `gDisplayTimeout` is uint8_t max 255; 255*1000=255000UL fits easily
    - Do NOT modify handleResetTopic() — it already resets to idle correctly (AC-5)
    - Do NOT modify NVS load/save — gDisplayTimeout already persisted correctly
