<objective>
## Goal
Tambahkan ESP32 task watchdog (30s timeout) dan MQTT offline indicator: ketika MQTT terputus, P2 baris 3 menampilkan "OFFLINE" dalam warna merah redup sebagai ganti status ANPR.

## Purpose
Parking gate display harus survive di lapangan tanpa pengawasan. Watchdog memastikan firmware restart jika hang. Offline indicator memberi tanda visual kepada operator bahwa koneksi terputus — tanpa mengganggu rendering panel lain.

## Output
- `esp_task_wdt` watchdog: init 30s di setup(), reset di loop() setiap iterasi
- `gMqttWasConnected` flag + `gMqttDisconnectedMs` timestamp untuk deteksi offline
- `renderPanels()`: jika MQTT offline >5s, P2 baris 3 tampil "OFFLINE" dim-red; else tetap anprStatus/"--"
</objective>

    - `true` in init means it will panic+reboot on timeout (desirable for field deployment)
    - Reset must be inside both the CALIBRATION_MODE early-return path and the normal path:
      add it just before BOTH `delay(5)` calls in loop()

<task type="auto">
  <name>Task 2: MQTT offline indicator on P2 row 3</name>
  <files>src/HD-WF1-WF2-LED-MatrixPanel-DMA.ino.cpp</files>
  <action>
    **Step 1: Add offline tracking globals**

    After `gLastEventMs` global (line ~111), add:
    ```cpp
    static bool          gMqttWasConnected    = false;   // tracks prior MQTT state
    static unsigned long gMqttDisconnectedMs  = 0;       // millis() when MQTT first lost
    ```

    **Step 2: Update connectivity tracking in loop()**

    In loop(), AFTER `connectMqttIfNeeded()` call and AFTER `if (mqttClient.connected()) { mqttClient.loop(); }`, add:
    ```cpp
    // Track MQTT connection state for offline indicator
    if (mqttClient.connected()) {
      gMqttWasConnected   = true;
      gMqttDisconnectedMs = 0;
    } else if (gMqttWasConnected && gMqttDisconnectedMs == 0) {
      gMqttDisconnectedMs = millis(); // record when connection was lost
      redrawNeeded = true;
    }
    ```

    **Step 3: Update renderPanels() P2 row 3**

    Replace the current P2 row 3 draw:
    ```cpp
    drawCenteredText(gGate.anprStatus.length() ? gGate.anprStatus : "--",
                     0, PANEL_RES_Y + 23, PANEL_RES_X, 8);
    ```
    With:
    ```cpp
    {
      const bool mqttOffline = gMqttWasConnected && gMqttDisconnectedMs > 0 &&
                               (millis() - gMqttDisconnectedMs >= 5000UL);
      if (mqttOffline) {
        targetSetTextColor(dma_display->color565(180, 0, 0)); // dim red
        drawCenteredText("OFFLINE", 0, PANEL_RES_Y + 23, PANEL_RES_X, 8);
      } else {
        targetSetTextColor(valueColor);
        drawCenteredText(gGate.anprStatus.length() ? gGate.anprStatus : "--",
                         0, PANEL_RES_Y + 23, PANEL_RES_X, 8);
      }
    }
    ```

    Notes:
    - `gMqttWasConnected` prevents showing "OFFLINE" on first boot before any connection
    - `gMqttDisconnectedMs == 0` when connected — the 5s check requires it to be non-zero
    - `redrawNeeded = true` in the disconnect detection ensures indicator appears promptly
    - Do NOT show "OFFLINE" if MQTT never connected (startup sequence)
    - The `targetSetTextColor(valueColor)` restore before the else branch — `valueColor` is
      defined earlier in `renderPanels()` as `dma_display->color565(0, 255, 80)`
  </action>

  </verify>
