<objective>
## Goal
Init NTP time sync via `configTime()` and maintain a global `gClockStr` string updated every second in `loop()`. When time is synced, string shows `"HH:MM:SS"` (WIB UTC+7); before sync it shows `"--:--:--"`.

## Purpose
Provides the clock data layer needed by Phase 2 Plan 02 (P2 rendering) and satisfies the project requirement of showing real-time clock on P2 row 2. Uses ESP32 built-in SNTP via `configTime()` — no new library dependency.

## Output
- `gClockStr` global (String) updated every second
- `updateClock()` function
- `configTime()` call in `setup()`
- Clock update + conditional `redrawNeeded = true` in `loop()`
</objective>

<task type="auto">
  <name>Task 2: Add updateClock() and wire into loop()</name>
  <files>src/HD-WF1-WF2-LED-MatrixPanel-DMA.ino.cpp</files>
  <action>
    1. Add `updateClock()` function before `loop()`. Place it in the "NTP / clock" section:
       ```cpp
       // Returns true if gClockStr changed (caller should set redrawNeeded)
       bool updateClock() {
         struct tm timeinfo;
         if (!getLocalTime(&timeinfo)) {
           // Not synced yet — leave gClockStr as "--:--:--"
           return false;
         }
         char buf[16];
         snprintf(buf, sizeof(buf), "%02d:%02d:%02d",
                  timeinfo.tm_hour, timeinfo.tm_min, timeinfo.tm_sec);
         if (gClockStr != buf) {
           gClockStr = buf;
           return true;
         }
         return false;
       }
       ```

    2. In `loop()`, after `mqttClient.loop()` and before the `redrawNeeded` check, add:
       ```cpp
       // Update clock every second
       const unsigned long nowMs = millis();
       if (nowMs - gLastClockUpdateMs >= 1000) {
         gLastClockUpdateMs = nowMs;
         if (updateClock()) {
           redrawNeeded = true;
         }
       }
       ```

    Avoid:
    - Do NOT use `delay()` in the clock update path.
    - Do NOT call `redrawNeeded = true` unconditionally — only when gClockStr actually changes (prevents spurious re-renders before sync).
    - The variable `nowMs` may conflict if `loop()` already defines it. If so, rename to `clockNowMs`.
  </action>

    - After WiFi connects and ~15 seconds elapse, Serial log shows renderPanels() being called each second.
    - Before WiFi, gClockStr remains "--:--:--" (visible in P2 after Plan 02 renders it).
  </verify>
  <done>AC-2 + AC-3 satisfied: clock updates every second when synced, "--:--:--" when not</done>
