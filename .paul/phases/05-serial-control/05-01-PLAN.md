    Di dalam `loop()`, tepat SEBELUM blok `if (redrawNeeded)`, tambahkan serial reader:
    ```cpp
    // Serial command input — format: cmd:json\n
    while (Serial.available()) {
      const char c = static_cast<char>(Serial.read());
      if (c == '\n') {
        gSerialBuffer.trim();
        if (gSerialBuffer.length() > 0) {
          const int colonIdx = gSerialBuffer.indexOf(':');
          if (colonIdx < 0) {
            Serial.println("[serial] invalid format, expected: <cmd>:<json>");
          } else {
            const String cmd     = gSerialBuffer.substring(0, colonIdx);
            const String payload = gSerialBuffer.substring(colonIdx + 1);
            if      (cmd == "event")    handleEventTopic(payload);
            else if (cmd == "greeting") handleGreetingTopic(payload);
            else if (cmd == "config")   handleConfigTopic(payload);
            else if (cmd == "reset")    handleResetTopic(payload);
            else Serial.printf("[serial] unknown command: %s\n", cmd.c_str());
          }
        }
        gSerialBuffer = "";
      } else if (c != '\r') {
        gSerialBuffer += c;
      }
    }
    ```

    JANGAN ubah Serial.begin(115200) di setup() — sudah ada.
    JANGAN ubah handler functions (handleEventTopic, dll) — digunakan bersama MQTT.

<task type="auto">
  <name>Task 2: Buat Python CLI tools/send_display.py</name>
  <files>tools/send_display.py</files>
  <action>
    Buat file `tools/send_display.py` dengan spesifikasi berikut:

    **Dependencies:** hanya `pyserial` (standard), `argparse` (builtin), `json` (builtin)

    **Auto-detect port:** coba `/dev/ttyUSB0`, `/dev/ttyUSB1`, `/dev/ttyACM0`, `/dev/ttyACM1` secara berurutan jika `--port` tidak diberikan.

    **Subcommands:**
    - `event` — kirim event akses kendaraan
      - `--status` (required): `granted` | `denied` | `scanning` | `idle`
      - `--plate` (optional, default ""): nomor plat, contoh "D 1234 ABC"
      - `--anpr` (optional, default ""): `detected` | `not_detected` | `low_confidence`
      - `--gate` (optional, default "Gate A"): nama gate
    - `greeting` — update teks P4
      - `--text` (required): teks greeting
      - `--color` (optional, default "white"): `white` | `yellow` | `cyan`
      - `--scroll` (optional, flag): aktifkan scroll
    - `config` — update konfigurasi
      - `--brightness` (optional, int 0-255)
      - `--timeout` (optional, int detik)
      - `--gate-name` (optional, string)
    - `reset` — reset display ke idle

    **Global options:**
    - `--port` (optional): serial port, contoh `/dev/ttyUSB0`
    - `--baud` (optional, default 115200): baud rate

    **Format yang dikirim:** `<cmd>:<json>\n` sesuai protokol firmware.

    **Output:** print ke stdout apa yang dikirim, contoh:
    ```
    Sending to /dev/ttyUSB0 (115200 baud):
    event:{"status":"granted","plate":"D 1234 ABC","anpr_status":"detected","gate_name":"Gate A","timestamp":0}
    OK
    ```

    **Error handling:** jika port tidak bisa dibuka, print pesan jelas dan exit(1).

    Tambahkan docstring di bagian atas file dengan contoh penggunaan lengkap.
  </action>
